# Budget

## C# conventions

- Do not suffix method names with `Async`, even for async methods (e.g. `Get()`, not `GetAsync()`). This applies to methods we write; third-party library APIs (e.g. `MongoDB.Driver`) keep their own naming.
- Do not default to `*OrDefault` methods (`FirstOrDefault`, `SingleOrDefault`, etc.) out of habit. Prefer throwing variants (`First`, `Single`) unless a missing result is a genuinely expected, legitimate case for that query.

## Blazor conventions

- Money amounts in the UI must always show exactly 2 digits after the decimal point (e.g. `29.98`, `50.00`, not `50`). Raw stored amount strings aren't guaranteed to have 2 decimals, so parse as `decimal` and format with `.ToString("0.00", CultureInfo.InvariantCulture)` rather than displaying the raw string.
- For any `@rendermode InteractiveAuto` page/component that fetches data in `OnInitializedAsync`, persist the fetched data via `[PersistentState]` properties and guard the fetch (e.g. `if (SomeProp is not null) { return; }`). Otherwise the component's `OnInitializedAsync` reruns and refetches everything a second time when the page hands off from interactive Server rendering to WebAssembly, causing a visible content "reload" flash. See `Transactions.razor`'s `TransactionList` and `Merchants.razor`'s `MerchantNames`/`Mappings` for the established pattern.
- Child components should not independently re-fetch reference data (e.g. merchant mappings/group assignments) that an ancestor `@rendermode InteractiveAuto` page already loads and persists via `[PersistentState]`. Pass the already-loaded data down as a `[Parameter]` instead. Otherwise the child's own `OnInitializedAsync` fetch reruns on the Server→WebAssembly handoff exactly like the unguarded-fetch case above, even though the parent did it right. See `CreditorSummary.razor`/`GroupSummary.razor` receiving `Mappings`/`GroupAssignments` from `Transactions.razor` for the established pattern.
- In SVG markup, never interpolate a raw `double` into a numeric attribute (`x`, `y`, `cx`, `cy`, viewBox, etc.) — Blazor's default `ToString()` uses the current culture, which may format decimals with a comma (e.g. `226,72486`), producing invalid SVG that the browser silently drops. Always format explicitly with `.ToString(CultureInfo.InvariantCulture)`.
- Never write an SVG `<text>` element directly in `.razor` markup. Razor's parser treats a bare `<text>` tag as its own "literal markup transition" syntax (used to emit plain text without a wrapping element), not as the SVG element — it silently emits no `<text>` DOM node at all, so any nested `<tspan>` renders nothing (a `<tspan>` outside a `<text>` parent is invalid SVG). Build SVG text as a raw HTML string (HTML-encode any interpolated content) and render it via `MarkupString` instead. See `Trends.razor`'s `RenderYAxisLabels`/`RenderXAxisLabels` for the established pattern.
